Team Page


Technical Description

Regarding the Html Techniques I used, semantic HTML tags like h1 tag for the main title and div tag for content structuring. Each member in our team is wrapped in a div class="team_Member" tag that shows the logical grouping of related content. I also added the a tag for easy navigation to individual student pages. Since the coursework PDF mentioned making interactive thumbnails accessible via keyboard, I added tabindex="0" for the team member containers.

The CSS techniques I used are Flexbox (display: flex; flex-wrap: wrap; justify-content: center;) for responsive and centered team member cards. Transition effects (transform: scale(1.1); opacity: 200;) for hover animations, adding visual feedback. Positioned the student details using position: absolute and controlled visibility through opacity and height transitions.


Accessibility

I have used alt attributes for images, like for example alt="student_1_image" to make sure that screen readers can describe images to visually impaired users. I added tabindex="0" for team members to access easily, so users can navigate the page without a mouse. The difference between text (color: white) and background (rgba(0, 0, 0, 0.7)) ensures readability for users with low vision. Additionally, I used clear, descriptive labels like "Role: Student 1" labels to help users understand each team member’s role.


Feedback Page


Technical Description

The HTML Techniques I used for the feedback page are to make a good structure for my feedback page. Firstly, I used semantic HTML elements like fieldset and legend tags to group personal and feedback-related inputs, ensuring better form organization. I applied label tag elements with for attributes linked to corresponding input tag fields for improved usability. I added a datalist for the "User's Interest Level in Clean Energy" input to offer auto-suggestions.


Accessibility

First of all, I used labels for every input field, helping screen readers identify elements. I added radio button groups with shared names like "feedback" and "satisfaction" to make sure that only one option can be selected at a time. I also included alt text for any future images like uploaded files and confirm button hover effects are visually clear.


Content Page


Technical Description

Regarding the content page, I used HTML Techniques such as adding semantic elements like h1, p, img, and div tags for structuring content. Flexbox is used for aligning author details and action icons such as like, share.

Regarding the CSS Techniques, I defined specific font sizes to 44px for titles, 18px for body text. I also added layouts like the main container, buttons, and for spacing, I used margins and padding to set a clean, spaced-out layout.


Accessibility

The content page has multiple accessibility features like semantic elements such as heading tags and paragraph tags, which give clear content structure and help screen readers navigate the page. Also, images include alt attributes that offer text descriptions. The page layout used ensures a logical visual flow. These aspects provide a basic level of accessibility for my content page.


Challenges and Lessons Learned


Until I finished the coursework, I faced different types of obstacles. One of them was searching through different free image downloading websites to find the most suitable images for the pages I had to make. It also took a lot of time for me to understand the codes and change them accordingly due to my lack of knowledge about CSS and JavaScript. However, I overcame this by learning those languages from YouTube and w3schools. I also struggled to finalize colors for the pages because my teammates had different opinions. Despite these problems, I successfully completed all tasks with the support and guidance of lecturers and team members.
